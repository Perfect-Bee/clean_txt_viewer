## 1. 프로젝트 소개

Clean TXT Viewer는 Flutter로 개발 중인 대용량 텍스트 뷰어입니다.

일반적인 텍스트 뷰어는 파일 전체를 메모리에 로드하기 때문에 수 MB 이상만 되어도 렉이 발생할 수 있습니다.

본 프로젝트는 TXT 파일 전체를 한 번에 읽지 않고, 필요한 부분만 페이지 단위로 읽어 화면에 표시하는 것을 목표로 합니다.

```text
- 대용량 TXT 파일 지원
- 페이지 단위 읽기
- 페이지 독립 렌더링
- 캐시 기반 페이지 이동
- 부드러운 탐색
- Android APK 배포
- 읽던 위치 저장 예정
```

장기적으로는 EasyViewer 수준의 TXT 리더를 목표로 합니다.

현재 단계에서는 자연스러운 무한 스크롤보다 대용량 TXT 파일을 안정적으로 처리하는 구조를 우선합니다.

## 2. 개발 목적

일반적인 방식은 TXT 파일 전체를 읽고 메모리에 적재한 뒤 UI에 렌더링합니다.

```text
TXT 파일
 ↓
전체 파일 읽기
 ↓
메모리 적재
 ↓
UI 렌더링
 ↓
렉 발생
```

Clean TXT Viewer는 현재 필요한 페이지 위치를 계산한 뒤, 필요한 부분만 읽고 캐시에 저장합니다.

```text
TXT 파일
 ↓
현재 페이지 위치 계산
 ↓
필요한 부분만 읽기
 ↓
현재 페이지 화면 표시
 ↓
주변 페이지 캐시 저장
```

따라서 수 MB 이상의 TXT 파일도 상대적으로 안정적으로 읽을 수 있습니다.

이번 수정 이후에는 화면에 읽은 모든 페이지를 누적하지 않고, 현재 페이지에 해당하는 텍스트만 유지하도록 변경했습니다.

```text
기존 방식
1페이지 + 2페이지 + 3페이지 + ... 계속 누적

수정 방식
현재 페이지 내용만 화면에 유지
```

이를 통해 스크롤을 오래 내릴수록 텍스트가 누적되어 버벅이는 문제를 줄이는 것을 목표로 합니다.

## 4. 핵심 실행 과정

Clean TXT Viewer의 핵심 실행 흐름은 다음과 같습니다.

```text
파일 선택
 ↓
파일 정보 생성
 ↓
현재 페이지 계산
 ↓
RandomAccessFile로 필요한 구간만 읽기
 ↓
UTF-8 안전 디코딩
 ↓
현재 페이지 텍스트 출력
 ↓
주변 페이지 캐시 유지
```

최근에는 연속 스크롤 누적 방식에서 페이지 독립 렌더링 방식으로 변경되었습니다.

```text
64KB
 ↓
현재 페이지 화면 출력
 ↓
스크롤 하단 감지
 ↓
다음 페이지 로딩
 ↓
기존 페이지 제거
 ↓
새 페이지로 화면 교체
```

결과적으로 사용자는 스크롤을 통해 다음 페이지로 이동할 수 있지만, 화면에는 항상 현재 페이지 분량만 유지됩니다.

이를 통해 대용량 TXT 파일을 오래 읽어도 텍스트가 계속 누적되지 않도록 개선했습니다.

## 5. 대용량 파일 처리 원리

기존 방식

```text
TXT
 ↓
전체 읽기
 ↓
메모리 적재
 ↓
화면 출력
```

문제

```text
파일 크기가 커질수록 렉 발생
```

현재 방식

```text
TXT
 ↓
현재 페이지 계산
 ↓
64KB 읽기
 ↓
텍스트 변환
 ↓
현재 페이지 화면 출력
```

예시

```text
10MB TXT

1페이지
64KB

2페이지
64KB

3페이지
64KB
```

필요한 부분만 읽습니다.

---

### 페이지 독립 처리

이번 수정 후 페이지는 서로 독립적으로 동작합니다.

```text
1페이지 화면
=
1페이지 텍스트만 보관

2페이지 화면
=
2페이지 텍스트만 보관

3페이지 화면
=
3페이지 텍스트만 보관
```

기존에는 다음 페이지를 읽을 때 기존 텍스트 뒤에 새 텍스트를 계속 붙였습니다.

```text
1페이지
 ↓
1페이지 + 2페이지
 ↓
1페이지 + 2페이지 + 3페이지
```

이 방식은 읽을수록 화면 데이터가 증가하기 때문에 대용량 TXT 처리에는 적합하지 않았습니다.

현재는 다음 페이지로 이동할 때 기존 페이지 텍스트를 제거하고 새 페이지 텍스트로 교체합니다.

```text
1페이지
 ↓
2페이지
 ↓
3페이지
```

이 방식은 무한 스크롤 느낌은 줄어들지만, 대용량 파일을 안정적으로 읽는 데 유리합니다.

---

### 캐시 처리

TxtPageReader는 페이지를 읽을 때 주변 페이지를 캐시에 저장합니다.

```text
이전 페이지
현재 페이지
다음 페이지
```

캐시는 페이지 이동 속도를 높이기 위한 보조 역할입니다.

화면에 표시되는 `_lines`는 현재 페이지 내용만 유지합니다.

```text
캐시
=
빠른 페이지 이동용

_lines
=
현재 화면 렌더링용
```

이 둘을 분리하여 화면 렌더링 데이터가 계속 증가하지 않도록 합니다.

### 스크롤 하단 자동 페이지 이동

현재 페이지의 하단 근처까지 스크롤하면 다음 페이지를 자동으로 불러옵니다.

```text

하단 근처 도달
 ↓
다음 페이지 로딩
 ↓
현재 화면을 다음 페이지로 교체
```

이전처럼 다음 페이지를 기존 텍스트 뒤에 붙이지 않습니다.

```text
다음 페이지
```

를 메모리에 유지합니다.

화면에 표시되는 줄 목록과 캐시는 분리되어 있습니다.

## 16. 최근 수정 사항

### 페이지 독립 렌더링 방식 도입

초기 연속 스크롤 버전은 스크롤 하단에 도달하면 다음 페이지를 자동으로 읽고, 기존 텍스트 뒤에 새 페이지 텍스트를 계속 추가하는 방식이었습니다.

```text
1페이지 로드
 ↓
스크롤 하단 도달
 ↓
2페이지 로드
 ↓
1페이지 + 2페이지 누적
 ↓
3페이지 로드
 ↓
1페이지 + 2페이지 + 3페이지 누적
```

이 방식은 사용자가 하나의 긴 문서를 읽는 느낌을 줄 수 있다는 장점이 있었지만, 대용량 TXT 파일 처리라는 프로젝트 목적과는 맞지 않는 문제가 있었습니다.

파일 자체는 64KB 단위로 나누어 읽고 있었지만, 화면에 표시되는 `_lines`에는 읽은 텍스트가 계속 누적되었습니다.

결과적으로 사용자가 스크롤을 많이 내릴수록 화면에 보관되는 텍스트 양이 증가했고, 대용량 파일을 오래 읽을수록 메모리 사용량과 렌더링 부담이 커지는 문제가 발생했습니다.

---

<details>
<summary>최근 수정 사항 상세 보기</summary>

### 변경 목적

이번 수정의 핵심 목적은 다음과 같습니다.

```text
대용량 TXT 파일을 읽을 때
화면에 현재 페이지 내용만 유지하도록 변경
```

기존 방식

```text
현재 페이지
+
다음 페이지
+
그 다음 페이지
+
계속 누적
```

문제점

```text
읽을수록 _lines 증가

ListView 렌더링 대상 증가

메모리 사용량 증가

장시간 읽기 시 버벅임 발생
```

수정 방식

```text
현재 페이지
만 유지
```

결과

```text
페이지가 바뀌면 이전 페이지 텍스트 제거

새 페이지 텍스트로 화면 교체

페이지끼리 서로 관여하지 않음

읽을수록 텍스트가 누적되지 않음
```

---

### 변경 전 구조

기존 `_loadNextPage()` 방식은 다음 페이지를 읽은 뒤 기존 `_lines` 뒤에 새 줄을 추가했습니다.

```dart
_lines = [
  ..._lines,
  ...newLines,
];
```

이 구조에서는 페이지를 넘길수록 화면 데이터가 계속 증가했습니다.

```text
1페이지
 ↓
1페이지 + 2페이지
 ↓
1페이지 + 2페이지 + 3페이지
 ↓
1페이지 + 2페이지 + 3페이지 + 4페이지
```

이 방식은 연속 스크롤처럼 보이지만, 대용량 TXT 처리에는 적합하지 않았습니다.

---

### 변경 후 구조

수정 후에는 다음 페이지로 넘어갈 때 기존 텍스트를 유지하지 않고, 현재 페이지 내용으로 교체합니다.

```dart
_lines = lines;
```

동작 방식은 다음과 같습니다.

```text
1페이지 화면
=
1페이지 텍스트만 보관

2페이지 화면
=
2페이지 텍스트만 보관

3페이지 화면
=
3페이지 텍스트만 보관
```

즉, 페이지가 서로 독립적으로 동작합니다.

---

### 스크롤 하단 감지 방식

스크롤이 현재 페이지 하단 근처에 도달하면 다음 페이지를 불러옵니다.

```dart
void _handleScroll() {
  if (!_scrollController.hasClients) return;

  final fileInfo = _fileInfo;
  if (fileInfo == null) return;

  if (_loading) return;
  if (_pageIndex >= fileInfo.pageCount - 1) return;

  final position = _scrollController.position;

  if (position.maxScrollExtent <= 0) return;

  if (position.pixels >= position.maxScrollExtent - 80) {
    _showPage(_pageIndex + 1, resetScroll: true);
  }
}
```

이전처럼 다음 페이지를 기존 화면에 붙이는 것이 아니라, `_showPage()`를 호출해 페이지를 교체합니다.

---

### 페이지 교체 방식

페이지를 읽는 핵심 함수는 `_showPage()`입니다.

```dart
final lines = await _pageReader.readPage(
  fileInfo: fileInfo,
  pageIndex: clamped,
);
```

읽은 결과는 기존 텍스트에 추가하지 않고, 현재 페이지 내용으로 교체합니다.

```dart
setState(() {
  _pageIndex = clamped;
  _previewPageIndex = clamped;
  _lines = lines;
  _loading = false;
});
```

이렇게 하면 화면에는 항상 현재 페이지에 해당하는 텍스트만 남습니다.

---

### 제거된 구조

이번 수정으로 다음 구조를 제거했습니다.

```text

_loadNextPage()

기존 페이지 + 다음 페이지 병합 방식
```

삭제 이유는 다음과 같습니다.

```text
대용량 TXT 처리 목적과 맞지 않음

읽을수록 화면 데이터가 증가함

캐시 제한 효과가 약해짐

장시간 사용 시 렌더링 부담 증가

```
